262 def flow(self, active):

264

266 content data. This is a simple flow-control mechanism that a 

267 peer can use to avoid oveflowing its queues or otherwise 

268 finding itself receiving more messages than it can process. 

269 Note that this method is not intended for window control. The 

270 peer that receives a request to stop sending content should 

271 finish sending the current content, if any, and then wait 

272 until it receives a Flow restart method. 

273 

274 RULE: 

275 

276 When a new channel is opened, it is active. Some 

277 applications assume that channels are inactive until 

278 started. To emulate this behaviour a client MAY open the 

279 channel, then pause it. 

280 

281 RULE: 

282 

283 When sending content data in multiple frames, a peer 

284 SHOULD monitor the channel for incoming methods and 

285 respond to a Channel.Flow as rapidly as possible. 

286 

287 RULE: 

288 

289 A peer MAY use the Channel.Flow method to throttle 

290 incoming content data for internal reasons, for example, 

291 when exchangeing data over a slower connection. 

292 

293 RULE: 

294 

295 The peer that requests a Channel.Flow method MAY 

296 disconnect and/or ban a peer that does not respect the 

297 request. 

298 

299 PARAMETERS: 

300 active: boolean

484 

485 This method creates an exchange if it does not already exist,

487 and expected class. 

488

490 

491 The server SHOULD support a minimum of 16 exchanges per 

492 virtual host and ideally, impose no limit except as 

493 defined by available resources.

497

509 

510 Each exchange belongs to one of a set of exchange 

511 types implemented by the server. The exchange types 

512 define the functionality of the exchange - i.e. how 

513 messages are routed through it. It is not valid or 

514 meaningful to attempt to change the type of an 

515 existing exchange. 

516 

517 RULE: 

518 

519 If the exchange already exists with a different 

520 type, the server MUST raise a connection exception 

521 with a reply code 507 (not allowed). 

522

538

541 reply code 404 (not found). 

542

546

552

554

559

562

564

2397 #
